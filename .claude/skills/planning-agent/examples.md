## Example 4: Planning System Migration

**Input:**
```
Plan: Migrate email system from Gmail to Microsoft 365
Team size: 25 people
Timeline: 1 month
Risk tolerance: Low (cannot afford downtime)
```

**Generated Plan:**

```markdown
---
status: draft
complexity: high
estimated_hours: 40
---

# Plan: Email System Migration to Microsoft 365

## Objective
Migrate 25 users from Gmail to Microsoft 365 with zero downtime over 1 month.

## Success Criteria
- [ ] All 25 users migrated successfully
- [ ] Zero email lost during migration
- [ ] All users trained on new system
- [ ] Old emails accessible in new system
- [ ] Downtime during migration < 1 hour per user

## Pre-Migration (Weeks 1-2)

### Week 1: Planning
- [ ] Audit current email usage (8 hrs)
- [ ] Identify migration tools (4 hrs)
- [ ] Create communication plan (2 hrs)
- [ ] Schedule migration windows (2 hrs)
- [ ] **Deliverable:** Migration plan document

### Week 2: Setup
- [ ] Set up Microsoft 365 tenant (4 hrs)
- [ ] Configure DNS and MX records (4 hrs)
- [ ] Test migration tools (4 hrs)
- [ ] Create user accounts (2 hrs)
- [ ] **Deliverable:** 365 environment ready

## Migration (Weeks 3-4)

### User Groups (5 users per day, 5 days)

**Monday-Friday:**
- [ ] Migrate Group 1 (5 users)
  - [ ] Backup existing emails (1 hr)
  - [ ] Migrate mailboxes (2 hrs)
  - [ ] Configure clients (1 hr)
  - [ ] Test send/receive (1 hr)
  - [ ] **Rollback plan:** Restore from backup if needed

- [ ] Migrate Group 2-5 (repeat pattern)
- [ ] Each group: 5 hours

### Post-Migration (Week 4)
- [ ] Verify all mailboxes migrated (4 hrs)
- [ ] Collect user feedback (2 hrs)
- [ ] Fix issues (4 hrs)
- [ ] Finalize documentation (2 hrs)

## Risk Mitigation
| Risk | Strategy |
|------|----------|
| Data loss | Triple backup: local, cloud, migration tool |
| Downtime | Migrate outside business hours |
| User resistance | Training, quick reference guides |
| Technical issues | Vendor support contract |

## Communication Plan
- **2 weeks before:** Announcement email
- **1 week before:** Training schedule
- **During migration:** Hourly status updates
- **After:** Support contact info

## Rollback Plan
If migration fails:
1. Restore from backup (1 hour)
2. Users continue with Gmail
3. Investigate failure
4. Reschedule migration
```
