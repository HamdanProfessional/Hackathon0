## Complexity Assessment Form

### Complexity Level: Low
- Clear objective
- Known approach
- Few dependencies
- Low risk

### Complexity Level: Medium
- Multiple components
- Some unknowns
- Several dependencies
- Moderate risk

### Complexity Level: High
- Ambiguous requirements
- Unknown approach
- Many dependencies
- High risk
- Multiple stakeholders

## Dependency Types

| Type | Description | Example |
|------|-------------|---------|
| **Sequential** | Must complete in order | Step 2 → Step 3 |
| **Parallel** | Can do simultaneously | Step A ⟂ Step B |
| **Blocking** | External dependency | Waiting for approval |
| **Resource** | Limited resource access | Server availability |
| **Skill** | Requires specific skillset | Need developer |

## Risk Assessment Matrix

| Risk | Likelihood | Impact | Priority |
|------|------------|--------|----------|
| Technical failure | Low | High | Plan B/C |
| Scope creep | Medium | Medium | Change control |
| Timeline delay | High | High | Buffer time |
| Resource shortage | Low | Medium | Cross-training |

## Priority Framework

### Eisenhower Matrix
- **Urgent + Important**: Do now
- **Urgent + Not Important**: Delegate
- **Not Urgent + Important**: Schedule
- **Not Urgent + Not Important**: Backlog

### MoSCoW Method
- **Must Have**: Critical for success
- **Should Have**: Important but not critical
- **Could Have**: Nice to have
- **Won't Have**: Out of scope

## Common Planning Patterns

### Problem-Solution-Result
1. **Problem**: What problem are we solving?
2. **Solution**: What's our approach?
3. **Result**: What success looks like

### MVP Approach
1. **Minimum**: What's the smallest viable version?
2. **Viable**: Does it solve the core problem?
3. **Product**: Can we ship and iterate?

### 5-Why Root Cause
1. Ask "why?" five times
2. Get to root cause
3. Address fundamental issue

### First Principles
1. Break down to assumptions
2. Validate each assumption
3. Build from facts
